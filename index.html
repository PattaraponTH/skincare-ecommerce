01 — Analysis & Design

การวิเคราะห์และออกแบบระบบ

ระบบร้านค้าออนไลน์สำหรับจำหน่ายผลิตภัณฑ์ Skincare ครบวงจร ประกอบด้วย 2 ส่วนหลัก ได้แก่ ฝั่งผู้ใช้งาน (User Frontend) สำหรับค้นหาและสั่งซื้อสินค้า และ ฝั่งผู้ดูแลระบบ (Admin Dashboard) สำหรับจัดการสินค้าและคำสั่งซื้อ

📋 Analysis — การวิเคราะห์ความต้องการ

ผู้ใช้งานระบบ (Stakeholders)

ประเภทผู้ใช้บทบาท
Guestเข้าชมสินค้า ค้นหาสินค้า แต่ยังไม่สามารถสั่งซื้อได้
Member (ลูกค้า)สมัครสมาชิก สั่งซื้อสินค้า ติดตามคำสั่งซื้อ
Adminจัดการสินค้า จัดการคำสั่งซื้อ ดู Dashboard รายงาน

ความต้องการฝั่งผู้ใช้งาน (User Side)

🔐 ระบบสมาชิก — สมัครสมาชิกด้วย Email/Password, เข้า-ออกระบบ, แก้ไขโปรไฟล์, รีเซ็ตรหัสผ่านผ่าน Email
🛒 ระบบสินค้า — แสดงรายการสินค้าพร้อมรูปภาพและราคา, กรองตามหมวดหมู่ (Cleanser, Moisturizer, Serum, Sunscreen), ค้นหาสินค้า, ดูรายละเอียด
🛍️ ตะกร้าสินค้า — เพิ่ม / ลบ / แก้ไขจำนวนสินค้าในตะกร้า, แสดงราคารวม Real-time, บันทึกตะกร้าไว้ครั้งต่อไป
💳 คำสั่งซื้อ — กรอกที่อยู่จัดส่ง, ยืนยันคำสั่งซื้อ, ดูประวัติคำสั่งซื้อและสถานะการจัดส่ง
💰 ชำระเงิน — เลือกวิธีชำระเงิน, บันทึก Transaction และสถานะการชำระ, เชื่อมโยงกับคำสั่งซื้อ
⭐ รีวิวสินค้า — ให้คะแนน 1–5 ดาว หลังซื้อสินค้า, แสดงความคิดเห็นและรีวิวจากผู้ซื้อ

ความต้องการฝั่งผู้ดูแลระบบ (Admin Side)

📊 Dashboard — ยอดขายรายวัน/รายเดือน, คำสั่งซื้อใหม่ที่รอดำเนินการ, สินค้าขายดี Top 5, จำนวนสมาชิกทั้งหมด
📦 จัดการสินค้า — เพิ่ม / แก้ไข / ลบสินค้า, จัดการหมวดหมู่, ตั้งราคาและสต็อก, ซ่อน / แสดงสินค้า
🧾 จัดการคำสั่งซื้อ — ดูรายการทั้งหมด, เปลี่ยนสถานะ (รอชำระ → ชำระแล้ว → จัดส่งแล้ว → สำเร็จ), ยกเลิกคำสั่งซื้อ
👥 จัดการสมาชิก — ดูรายชื่อสมาชิก, ระงับ / ปลดระงับบัญชี, ดูประวัติการสั่งซื้อของแต่ละสมาชิก
🎨 Design — การออกแบบระบบ

สถาปัตยกรรมระบบ (System Architecture)

ระบบใช้ 3-Layer Architecture แบ่งเป็น Frontend, Backend และ Database สื่อสารผ่าน REST API

4.1 Frontend

React.jsCSS3SPA

แบ่งหน้าตาม Route: /, /products, /cart, /checkout, /admin — ใช้ JWT Token เก็บใน localStorage

4.2 Backend

Node.jsExpress.jsJWT

Module: User, Product, Order, Payment, Review — แยกตาม Separation of Concerns

4.3 Database

PostgreSQL

ตารางหลัก: users, products, categories, orders, order_items, payments, reviews

Database Design

ตารางคำอธิบายคอลัมน์สำคัญ
usersข้อมูลผู้ใช้งานและการยืนยันตัวตนid, name, email, password, role
productsข้อมูลสินค้า ราคา สต็อกid, name, price, stock, category_id
categoriesหมวดหมู่สินค้า (เซรั่ม, ครีมกันแดด ฯลฯ)id, name
ordersคำสั่งซื้อและสถานะid, user_id, total, status
order_itemsรายการสินค้าในแต่ละคำสั่งซื้อid, order_id, product_id, qty, price
paymentsข้อมูลการชำระเงินและ transactionid, order_id, method, status, paid_at
reviewsรีวิวและคะแนนสินค้าจากผู้ซื้อid, user_id, product_id, rating, comment

Security Design

Authentication — JWT (JSON Web Token) หมดอายุใน 24 ชั่วโมง
Authorization — แยก Role: member และ admin
Password — เข้ารหัสด้วย bcrypt
HTTPS — บังคับใช้ใน Production
Input Validation — ตรวจสอบข้อมูลก่อน Query ทุกครั้ง

Software Architecture Principles

หลักการการนำไปใช้ในโปรเจกต์นี้
Separation of Concernsแยก Frontend / Backend / Database ออกจากกันชัดเจน
Single Responsibilityแต่ละ Module รับผิดชอบเพียงหน้าที่เดียว เช่น OrderModule ดูแลเฉพาะคำสั่งซื้อ
Modularityแบ่ง Backend เป็น User, Product, Order, Payment, Review Module
Loose CouplingFrontend และ Backend สื่อสารผ่าน REST API เท่านั้น
Security by DesignJWT Authentication + bcrypt + Input Validation ทุกจุด
